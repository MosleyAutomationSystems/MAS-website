The framework

The Mosley Standard — v2.1

Four pillars, equal weight. A failure in one degrades the integrity of the whole. Every MAS project is checked against all four before it's called done.

Pillar I

Infrastructure

The foundation. Environments are reproducible from source, dependencies are pinned and explicit, and configuration lives in version control — not only in a running system.

Pillar II

Operations

The execution. Anything done more than twice is a candidate for automation, every automated action has a defined failure path, and nothing touches production without a rollback plan.

Pillar III

Accessibility

Universal access. Semantic HTML, mobile-first CSS Grid, full keyboard and screen-reader operability. Treated as an operational requirement, not a design nicety.

Pillar IV

Security

The integrity. Least privilege by default, secrets never in version control, every automated action logged. Verified against seven referenced frameworks — CERT, CWE, CVE, OWASP, DISA STIG, NVD, and CVSS.

The Mosley Standard is a personal engineering framework, not a regulatory standard or a substitute for a formal third-party audit. Passing it means a project meets MAS's own bar — a bar written down, versioned, and applied the same way to every client.
